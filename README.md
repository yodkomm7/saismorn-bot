# LINE Bot: ผู้ช่วยคู่คิดเพื่อนยาก (Bill Splitter & Rebalancing Bot)

ระบบแชทบอท LINE สำหรับกลุ่มเพื่อน เพื่อช่วยจำเลขบัญชีของเพื่อนแต่ละคน และคำนวณหารค่าอาหาร/เคลียร์ยอดรายจ่ายของกลุ่มเพื่อนอย่างยุติธรรมและรวดเร็ว

## 🚀 ฟังก์ชันหลักของระบบ (Core Features)

1. **บันทึกบัญชีของตนเอง**: เพื่อนในกลุ่มสามารถบันทึกข้อมูลธนาคาร หรือเบอร์พร้อมเพย์เก็บไว้ในระบบได้
2. **เรียกดูบัญชีธนาคาร**: สามารถเรียกดูข้อมูลบัญชีของทุกคนในกลุ่มด้วยรูปแบบ **Flex Message Card** ที่สวยงาม พร้อมปุ่ม **"คัดลอกเลขบัญชี" (Copy)** ไปวางโอนได้ทันที
3. **หารค่าอาหารเท่ากัน (Equal Split)**: สร้างบิลที่คนจ่ายมีคนเดียวและเพื่อนๆ หารแบ่งจ่ายเท่ากันทุกคน
4. **เฉลี่ยยอดทริป/ปาร์ตี้ (Multi-Payer Rebalancing)**: กรณีที่ทุกคนช่วยกันออกเงินคนละอย่าง (เช่น คนนึงออกค่ารถ อีกคนออกค่าอาหาร อีกคนออกค่าขนม) บอทจะทำการเฉลี่ยยอดรวมทั้งหมด และคำนวณรายการโอนคืนแบบลดจำนวนการทำธุรกรรมให้น้อยที่สุด (Greedy Settlement Algorithm) พร้อมแสดงเลขบัญชีของผู้รับโอนที่ถูกต้องโดยอัตโนมัติ

## 🛠️ ขั้นตอนการติดตั้งและเริ่มใช้งาน (Setup Guide)

### 1. ติดตั้ง Node.js Dependencies
ทำการตรวจสอบให้แน่ใจว่าติดตั้ง Node.js เรียบร้อยแล้ว จากนั้นรันคำสั่ง:
```bash
npm install
```

### 2. ตั้งค่าไฟล์สภาพแวดล้อม (Environment Config)
คัดลอกไฟล์ `.env.example` ไปเป็น `.env` และกรอกค่าตามที่สร้างใน LINE Developer Console:
```bash
cp .env.example .env
```
ภายในไฟล์ `.env` จะมีรายละเอียดดังนี้:
```env
PORT=3000
LINE_CHANNEL_ACCESS_TOKEN=ใส่_Channel_Access_Token_ของคุณตรงนี้
LINE_CHANNEL_SECRET=ใส่_Channel_Secret_ของคุณตรงนี้
GEMINI_API_KEY=ใส่_Gemini_API_Key_ของคุณตรงนี้

# Postgres connection string (จาก Neon)
DATABASE_URL=ใส่ connection string จาก Neon ตรงนี้
```

### ตั้งค่า Postgres Database (Neon)

ระบบเก็บข้อมูลผู้ใช้/บิลทั้งหมดบน Postgres จริง (ไม่ใช่ไฟล์ local อีกต่อไป) แนะนำใช้ [Neon](https://neon.tech) เพราะมี free tier ที่ไม่ลบข้อมูลทิ้ง (แค่ auto-suspend ตอนไม่มีคนใช้ แล้ว resume เองในเสี้ยววินาทีตอนมี query เข้ามา):

1. สมัครบัญชีที่ [neon.tech](https://neon.tech) (ฟรี ไม่ต้องผูกบัตรเครดิต)
2. สร้าง Project ใหม่ (เลือก region ใกล้ๆ เช่น Singapore)
3. ในหน้า Dashboard ของ Project กด **Connection string** แล้วคัดลอกมาใส่ในตัวแปร `DATABASE_URL` ของไฟล์ `.env`
4. รันคำสั่งนี้ครั้งเดียวเพื่อสร้างตาราง + ย้ายข้อมูลเดิมจาก `db.json` (ถ้ามี) เข้า Postgres:
   ```bash
   node src/scratch/migrateToPostgres.js
   ```
5. รัน server ตามปกติ (`npm run dev`) — ตารางจะถูกสร้างอัตโนมัติถ้ายังไม่มีอยู่แล้ว

> **หมายเหตุความปลอดภัย:** `DATABASE_URL` มี password ของฐานข้อมูลฝังอยู่ ต้องเก็บไว้ใน `.env` เท่านั้น (ห้าม commit เข้า git) — Neon เข้ารหัสการเชื่อมต่อด้วย TLS และมี authentication ในตัวอยู่แล้ว ต่างจาก cloud bin สาธารณะแบบเดิมที่ไม่มี auth ป้องกัน

### 3. รัน Server ในโหมดพัฒนา (Development Mode)
คุณสามารถรัน Server ได้ผ่านคำสั่ง:
```bash
npm run dev
```
เซิร์ฟเวอร์จะรันที่พอร์ต `3000` และเปิดใช้งาน Webhook Endpoint ที่: `http://localhost:3000/webhook`

### 4. เปิดใช้งาน Webhook สาธารณะด้วย ngrok
เนื่องจาก LINE ต้องส่งข้อมูล Webhook ไปยัง URL ที่เป็น `https` และเข้าถึงได้จากอินเทอร์เน็ตภายนอก คุณสามารถใช้ `ngrok` เพื่อทำ Tunnel ได้โดยรัน:
```bash
ngrok http 3000
```
หลังจากรันแล้ว ngrok จะให้ Forwarding URL ที่หน้าตาคล้าย ๆ แบบนี้:
`https://xxxx-xx-xx-xx.ngrok-free.app`

ให้นำ URL นี้ไปกรอกในหน้า **LINE Developers Console** ที่หัวข้อ **Webhook URL** โดยต้องเติมพาธ `/webhook` เข้าไปด้วย เช่น:
`https://xxxx-xx-xx-xx.ngrok-free.app/webhook`

## 💬 รายการคำสั่งในแชท (Chat Commands)

สามารถพิมพ์คำสั่งเหล่านี้ในกลุ่ม LINE เพื่อควบคุมบอทได้ทันที:

### 1. จัดการข้อมูลธนาคาร
* **บันทึกบัญชี [ธนาคาร] [เลขบัญชี] [ชื่อเจ้าของบัญชี]**
  * *ตัวอย่าง:* `บันทึกบัญชี กสิกร 123-4-56789-0 สมชาย ใจดี`
  * *ตัวอย่าง (พร้อมเพย์):* `บันทึก พร้อมเพย์ 0891234567 สมศรี ดีงาม`
* **ดูบัญชี** (หรือ `/accounts`)
  * บอทจะส่งลิสต์การ์ดบัญชีของทุกคนที่เคยลงทะเบียนในกลุ่ม สามารถกดปุ่มคัดลอกเลขบัญชีเพื่อนำไปโอนเงินได้

### 2. หารค่าอาหาร (หารเท่ากันปกติ)
* **หารเท่ากัน [ยอดรวม] [ชื่อมื้ออาหาร]**
  * *ตัวอย่าง:* `หารเท่ากัน 1200 ค่าชาบู`
  * บอทจะเปิดบิล ให้เพื่อนๆ กดปุ่ม **"🙋‍♂️ เข้าร่วมหาร"**
  * เมื่อลงชื่อครบแล้ว ให้พิมพ์คำสั่ง **"สรุปยอด"** หรือกดปุ่มสรุปยอด บอทจะส่งข้อมูลการโอนเงินคืนและเลขบัญชีของผู้จ่ายเงิน

### 3. เฉลี่ยค่าใช้จ่ายรวมหลายคน (ปาร์ตี้/ทริป)
* **เริ่มเฉลี่ย [ชื่อทริป/ปาร์ตี้]**
  * *ตัวอย่าง:* `เริ่มเฉลี่ย ทริปพัทยา`
  * บอทจะสร้างห้องจัดเก็บยอดรายจ่ายของทริปนั้นๆ
* **จ่าย [ยอดเงิน] [ชื่อรายการ]**
  * *ตัวอย่าง:* `จ่าย 800 ค่าเค้ก`
  * *ตัวอย่าง:* `จ่าย 1500 ค่าอาหารเย็น`
  * สมาชิกแต่ละคนที่ออกเงินไปก่อน สามารถพิมพ์ส่งยอดเงินเพื่อบันทึกในทริป บอทจะคอยสรุปและแสดงรายการให้ดูบน Flex Message ตลอด
* **สรุปยอด**
  * พิมพ์เมื่อสมาชิกลงยอดค่าใช้จ่ายในกลุ่มครบทั้งหมดแล้ว บอทจะคำนวณและแสดงขั้นตอนการโอนเงินคืนแบบสุทธิทันที (คนจ่ายน้อยกว่าเฉลี่ย โอนหา คนจ่ายมากกว่าเฉลี่ย พร้อมแสดงบัญชีผู้รับโอน)
* **ยกเลิกปาร์ตี้**
  * ยกเลิกข้อมูลทริป/ปาร์ตี้ปัจจุบัน
